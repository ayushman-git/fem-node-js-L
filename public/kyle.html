Getting started with JS

  • The '+' operater can be defined as overlaoded because it can perform two different operations based on the operands.
    • Mathematical addition - 4 + 2
    • String concatenation - "Hello" + " World"
  • '==' is also called loose equality operator. It allows coercion if possible.
  • NULL is an empty value but it is an object.
  • Array is basically a subset of object. That's why typeof [2,3] will return object.
  • The block of code which checks the logic condition is known as Test Clause
  • arr.pop() deletes an array from last element. arr.shift() deletes an array from first element.
  • Primitive data types: - undefined, string, number, symbol, object, boolean
  • null is defined as an object, and so do function and arrays. They are subset of object.
  • In JS we can define one type of var with another type of value because JS is not strongly typed.
    So let v = 2; v = "string"; v = {a: "A"}
  • Coercion means converting one data type to another.
  • Implicit coercion means converting one data type into another but doing while making a copy and not changing the actual value in the variable.
  • Falsey value the outputs which will be turned into false after coercing it to a boolean value
  • Function expressions means assigning a funcition to a variable which may be then passed as argument in another funciton.
  • IIFE - Immediately Invoked Function Expression, they can be used to create new variable with same name as a global variable and not changing its name. They are also executed as soon as they are declared.
  • Explicit let blocks - these are blocks which can be created anywhere to create a let variable only for that block.
  • Prototypes are the mechanism by which JavaScript objects inherit features from one another.
  • Class - Class can be used to create several instance of a prticular group of data.
