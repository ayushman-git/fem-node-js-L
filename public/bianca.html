From Fundamental to Functional

  • '.' operater is always used with objects.
  • Reference Value
    Non-primitive (objects) values are stored as reference. Primite values are stored as values.
    Values stored in objects are actually pointers that reference to the value. Obj -> name -> "Ayushman" Primitive values directly have the value in that variable. sameName -> "Ayushman"
  • '.' notation coerce the variable intro string except when its starting with digits or dash.
  • The main difference between array and object is that array has order because of sequeantial indices. So we can use multiple built in properties and functions.
  • With the help of destructuring, we can assign values to new variables from a source (obj, array etc) in a single line.
  • forEach iterates through each element but don't return anything. On the other hand map iterates as well as return a new array.

Arrow Function

  • The main difference between arrow and normal function is of 'this' meaning
  • In a normal function every function have their own this reference.
  • But arrow functions don't have their own this reference. So they can be used to refer to the parent function.
  • Projection means taking data from one data structure and converting it into another one.
  • Reduce() - It takes and array and return a single output.
    arr.reduce(callback( accumulator, currentValue[, index[, array]] )[, initialValue])
  • Currying - it means we can give arguments to a function at different time and once function has all the arguments it will be executed.
  • Composing - it means combining several functions into one function.
  • Closure - A closure is the combination of a function bundled together (enclosed) with references to its surrounding state (the lexical environment).
